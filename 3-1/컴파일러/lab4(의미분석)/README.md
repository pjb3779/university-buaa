의미 분석
해당 과제에서는 심볼들을 특정하여 해당 심볼이 어떤 스코프에 속해있는지 심볼 테이블에 저장한후에 출력하는 과제입니다. 또한 나머지 에러처리도 구현하는 과제입니다.

심볼 관리는 사실 그렇게 어렵지 않습니다. 심볼을 추천하는것은 ident토큰을 사용하여 추출하면됩니다. 하지만 스코프 관리를 하는것이 관건입니다. 저는 기본적으로는 블럭을 가지고 스코프 처리를 한후에 전역 스코프와 대조하여 현재 스코프가 몇인지를 알수 있는방식으로 설계 하였습니다.
본 과제에서는 사실 에러처리가 가장 어려웟었는데 에러 E를 처리할떄 많은 생각이 필요합니다. 

주의 사항
구현하는것은 생각보다 얼마 걸리지 않았으나 에러 e를 처리함에 있어서 많은 시도를 해보았습니다. 그래서 코드를 한번 뒤집어 엎은적도 있습니다. 다시 한번 첫 설계의 중요성을 깨달았습니다.. 
그리고 c++언어로 구현을 하였는데 왜 객체지향 설계를 하지 않았나 라는 후회를 했습니다. 너무 자바 = 객체지향 이라는 사고방식에 갇혀 있지 않았나 하고 생각했습니다.
